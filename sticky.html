Fixed Positioning: When to Use

  • Keep headers, footers, or navbars visible at all times.
  • Floating action buttons or “back to top” widgets.
  • Persistent chat/support icons.

Fixed Positioning: When Not to Use

  • Large layout containers that should scroll with content.
  • Elements needing to participate in the normal flow.

Fixed Positioning: Precautions

  • Manage z-index so fixed elements don’t hide critical UI.
  • Ensure fixed items are sized for mobile screens.
  • Use sparingly—too many fixed items can clutter the page.

Sticky Positioning: When to Use

  • Keep section headers or sub-navigation in view as you scroll.
  • Ideal for table headers within a scrollable panel.
  • Use for in-page anchors and tab menus.

Sticky Positioning: When Not to Use

  • For elements that must remain visible outside their container’s scroll area.
  • When the container isn’t scrollable or has no height limit.

Sticky Positioning: Precautions

  • Requires a scrollable ancestor with a fixed height or max-height.
  • May not work in very old browsers—always test.
  • Combine with sensible top offsets to avoid hiding content.
